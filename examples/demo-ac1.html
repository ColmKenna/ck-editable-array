ck-editable-array – Basic CRUD Operations Demo

This demo demonstrates basic Create, Read, Update, Delete (CRUD) operations using the ck-editable-array component. It shows how to provide light-DOM templates for slot="display" and slot="edit", set the data property, handle form submissions, and inspect component state in real-time.

Try it: Click "Add" to create new rows, click "Edit" to modify existing data, or click "Delete" to remove rows. Watch the data inspection panel update in real-time!

1. Default Add Button with Data Inspection

This example uses the default Add button (no custom template). The component has a name="addresses" attribute, which means when used in a form, the data will be submitted as a field named "addresses".

2. Custom Add Button with Different Data Types

3. Readonly Mode

This example demonstrates readonly mode. When the readonly attribute is set, all edit controls (Add, Edit, Delete buttons) are disabled. This is useful for displaying data that shouldn't be modified.

📤 Form Submission Example

The ck-editable-array component integrates seamlessly with HTML forms. When placed inside a form, the component's data is automatically included in form submissions. The name attribute determines the field name (e.g., "addresses", "contacts", "items").

Form Field Name Demonstration:

  • Demo 1 uses name="addresses" → Form data will include an "addresses" field
  • Demo 2 uses name="contacts" → Form data will include a "contacts" field
  • Demo 3 uses name="items" → Form data will include an "items" field
