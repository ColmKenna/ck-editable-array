HTML Template Code 
<ck-editable-array id="demo1" name="contact">
  <template slot="display">
    <div class="row-display">
      <div class="content">
        <strong><span data-bind="name"></span></strong>
        <span> — </span>
        <span data-bind="email"></span>
      </div>
      <div class="actions">
        <button data-action="toggle">Edit</button>
        <button data-action="delete">Delete</button>
        <button data-action="restore">Restore</button>
      </div>
    </div>
  </template>

  <template slot="edit">
    <div class="row-edit">
      <div class="form-field">
        <label>Name</label>
        <input data-bind="name" placeholder="Enter name" />
      </div>
      <div class="form-field">
        <label>Email</label>
        <input data-bind="email" placeholder="Enter email" />
      </div>
      <div>
        <button data-action="save">Save</button>
        <button data-action="cancel">Cancel</button>
      </div>
    </div>
  </template>
</ck-editable-array>

<script>
  // Set initial data
  demo1.data = [
    { name: 'Alice Smith', email: 'alice@example.com' },
    { name: 'Bob Jones', email: 'bob@example.com' }
  ];

  // Listen for data changes
  demo1.addEventListener('datachanged', (e) => {
    console.log('Data changed:', e.detail.data);
  });
</script>

✅ Feature 2: Schema-Driven Validation

Validation Accessibility

Try this: Click "Edit" on the empty row. Notice the Save button is disabled, error messages appear, and the row has a red border. Fill in the fields to see errors clear in real-time. Screen readers will announce errors via ARIA live regions (role="alert", aria-live="polite"). The component uses JSON Schema for validation with required and minLength constraints.

Validation Schema Setup 
// Define JSON Schema for validation
demo2.schema = {
  type: 'object',
  properties: {
    name: { type: 'string', minLength: 1 },
    email: { type: 'string', minLength: 1 }
  },
  required: ['name', 'email']
};

// Set initial data (one valid, one invalid)
demo2.data = [
  { name: 'Alice Smith', email: 'alice@example.com' },
  { name: '', email: '' }  // This row will show validation errors
];

🔗 Feature 3: Nested Property Binding

Core

Try this: Edit a row to see nested property paths in action. The component supports dot notation like data-bind="person.name" and data-bind="address.city" for complex data structures. This allows you to work with deeply nested objects without flattening your data model.

Nested Data Structure 
// Complex nested data structure

🔒 Feature 4: Exclusive Locking

Core

Try this: Click "Edit" on any row. Notice that other rows become locked (grayed out and disabled with data-locked attribute) and the Add button is disabled. Only one row can be edited at a time, preventing data conflicts. The component uses the inert attribute for accessibility compliance.

Exclusive Locking Behavior 
// When one row enters edit mode:
// 1. All other rows get [data-locked] attribute
// 2. Add button becomes disabled
// 3. Locked rows are inert (not keyboard navigable)

// Listen for mode changes
demo4.addEventListener('aftertogglemode', (e) => {
  console.log('Mode changed:', e.detail);
  // detail.mode: 'display' or 'edit'
  // detail.index: row index
});

// The component automatically manages locking
// No manual intervention required!

🎨 Feature 5: Custom Styling with CSS Parts

Theming

Try this: This example uses custom CSS injected via the slot="styles" to create a unique look with gradient backgrounds. The component provides CSS parts, data attributes, and classes for complete styling control. All styles are scoped to the shadow DOM, preventing conflicts.

Custom Styling Example 
<ck-editable-array id="demo5">
  <!-- Templates omitted for brevity -->
  
  <style slot="styles">
    /* Styles injected into shadow DOM */
    .row-display {
      background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
      color: white;
      padding: 16px;
      border-radius: 8px;
    }
    
    .row-edit {
      background: linear-gradient(135deg, #f093fb 0%, #f5576c 100%);
      border: none;
      color: white;
    }
    
    /* Target specific elements with classes */
    .row-display button {
      background: rgba(255, 255, 255, 0.2);
    }
  </style>
</ck-editable-array>

📊 Feature 6: Readonly Mode

Core

Try this: This array is in readonly mode via the readonly attribute. Notice that the Add button is disabled, and all inputs are read-only. This is useful for displaying data that shouldn't be edited, such as historical records or audit logs. Users can still view details but cannot make changes.

Readonly Mode Usage 
<!-- Add readonly attribute to enable readonly mode -->
<ck-editable-array id="demo6" readonly>
  <!-- Templates remain the same -->
</ck-editable-array>

<script>
  // Set data as usual
  demo6.data = [
    { label: 'This is read-only data' },
    { label: 'You can view but not edit' }
  ];
  
  // Toggle readonly mode programmatically
  demo6.readonly = false;  // Enable editing
  demo6.readonly = true;   // Disable editing
  
  // Check readonly state
  console.log(demo6.readonly);  // true or false
</script>
