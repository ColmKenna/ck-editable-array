🔍 Validation & Accessibility Demo

♿ Accessibility Features

  • ARIA Invalid: Invalid fields have aria-invalid="true"
  • Error Association: Error messages linked via aria-describedby
  • Live Regions: Error summary uses role="alert" and aria-live="polite"
  • Visual Indicators: Invalid rows and fields are clearly marked
  • Keyboard Support: All validation works with keyboard-only navigation

Example 1: Required Fields

Try clicking "Edit" on the empty row. Notice the Save button is disabled, error messages appear, and the row has a red border. Fill in the fields to see errors clear.

Example 2: Partial Validation

This example shows how validation works when only some fields are invalid. Try editing the second row and clearing just the email field.

Example 3: Add New Item with Validation

Click "Add Contact" to create a new row. It starts in edit mode with empty fields, showing validation errors immediately. The Save button is disabled until all required fields are filled.
