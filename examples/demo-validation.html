🔍 Validation & Accessibility Demo

♿ Accessibility Features

  • ARIA Invalid: Invalid fields have aria-invalid="true"
  • Error Association: Error messages linked via aria-describedby
  • Live Regions: Error summary uses role="alert" and aria-live="polite"
  • Visual Indicators: Invalid rows and fields are clearly marked
  • Keyboard Support: All validation works with keyboard-only navigation

Screen Reader Testing Tips

  1. Turn on a screen reader (NVDA/JAWS on Windows, VoiceOver on macOS).
  2. Activate an "Edit" button and navigate through inputs with Tab/Shift+Tab.
  3. Leave a required field empty to hear aria-invalid announced.
  4. Listen for the error summary live region after edits. It should read out changes.
  5. Confirm that error text is associated to inputs via aria-describedby.
